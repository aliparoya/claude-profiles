# Style Guide

This is the reference for anyone producing material that carries the project's name. Blog posts, diagrams, decks, hero images, Confluence pages, internal write-ups. The look and the language sit here.

Use the guide before you start, not after. Pick the page that matches what you are about to make, skim it, and then write.

## Pages

- [Brand](./brand): palette, typography, logo, and the broad aesthetic the project leans on.
- [Writing](./writing): voice, tone, and the rules for prose that does not read like a press release.
- [Images](./images): when a photograph fits, when an illustration fits, what to avoid.
- [Image prompts](./image-prompts): prompt patterns for producing imagery.
- [Diagrams](./diagrams): when a diagram earns its space, and what shape it should take.
- [Presentations](./presentations): when a deck is the right format, and when it is not.
- [MDX reference](./mdx-reference): copy-paste snippets for non-obvious MDX patterns.
